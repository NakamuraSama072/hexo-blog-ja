2025-8-9 v1.0 Changelog

这是自建站以来的第一篇博客。由于建站过程很累，而且中途差错不断，我就简单说几点网站内容和遇到的困难，就权且当做是一个网站测试吧。

之后每次大版本更新完都会发布一个Changelog，以展示新功能、错误修复和一些细节上的优化。

（你们估计要吐槽白天模式下中间的字看不清楚，是的，我一直在改背景图片的曝光度，但是改来改去也就这个样，很难达到夜晚模式下的效果，所以干脆直接摆烂了。另外，伊吹酱可爱捏）

新功能/What’s New

新增特点/New Features

  • 使用 Hexo 7.3.0 搭建博客框架，以及 Redefine 2.8.5 作为主题。

  • 新增了“分类”、“友情链接”和“标签”页面，方便查找博客内容以及列出参考的其他博客网站等。别问我为什么加顿号

错误修复/Bug Fix

  • 修复在 Firefox 内网页渲染异常错误的问题
  • 修复在“链接”子菜单内打开相应链接时页面提示 404 的问题

关于博客

这个博客是干什么的？

就如网站横幅标题所要展现的，“Linux and You”。因此，这个博客主要是一些关于Linux的内容，当然也有可能会随机刷新诸如C/C++或者Python之类的文章。

你是怎么搭建起这个博客的？

一开始我用的是 ClawCloud 的云服务（毕竟它的价格非常诱人，Github 老用户还送 $5 免费额度），但是在过了两天之后，博客文件所在的 pod 一直显示Pending（崩了），重启数次也无果（白忙了 2 天……），于是一怒之下直接删库跑路。

在对比了多家云服务商后，我发现 Azure 和 AWS 费用高的吓人，界面也很难操作；Google 的费用相对低很多，但是有着潜在的隐私问题——你也不想你的个人信息被一家广告商拿去用来盈利吧。

最后，我决定考虑Github。由于Vercel在某些国家/地区被墙（你可以用某些特殊手段），我采用Github和Netlify的方式部署，目前一直没出什么差错。

建站时出现过哪些问题？

除了之前在“错误修复”和上面部分提到的问题以外，还出现过以下问题：

  1. 分支填错。初始分支是main，我记成了master（后来被我删了），导致 VSCode 内 Git 的master分支像幽灵一样一直跑不掉，最后只能删库从头再来。（好在还没有做什么配置）
  2. 中日韩文字乱码，甚至渲染异常。找了一圈最后才发现是我 VSCode 用的是 GBK 而不是 UTF-8 编码导致的。

其他的一些次要问题和不可抗力因素（比如说上课），这里就不一一展开了。

博客将会采用哪种更新计划？

博客将采用类似Debian和Ubuntu的稳定版本更新策略，而非滚动更新。如果采用滚动更新模式，可能会导致以下问题：

  • Git 提交过于频繁，使得版本历史杂乱，难以管理。
  • 难以敲定新版本的功能，因为持续的小幅更新可能缺乏明确的阶段性目标。
  • 时间问题，频繁更新会增加维护成本，而稳定版本能更好地平衡更新节奏与内容质量。

因此，博客会不定期发布经过测试和完善的版本，确保每次更新都带来稳定的体验。

后续的新功能

  • 新增“关于”页面
  • 在底部增加文章推荐部分以快速跳转
  • 重新整理导航栏的内容
  • 增加评论功能 （目前还没有计划）
